# 參數表使用說明（Excel → CSV → 遊戲）

## 檔案位置

| 檔案 | 路徑 | 角色 |
|---|---|---|
| **Excel 主檔** | `config/Excel/game_parameters.xlsx` | 你平常編輯的地方（工作表：`game_parameters`） |
| CSV（自動產生） | `config/CSV/game_parameters.csv` | 由 xlsx 轉出、系統實際讀取；**每次套用會被覆蓋，不用手改** |

## 怎麼調整遊戲數值（雙擊一次就好）

1. 用 **Excel 打開 `config/Excel/game_parameters.xlsx`**，在 `game_parameters` 工作表改「參數a、參數b、…」欄裡的**數字**（或元組裡的數字）。
2. **存檔（Ctrl+S）**。此時遊戲**不會**刷新。（Excel 可以繼續開著，不必關。）
3. **雙擊專案根目錄的「套用參數.bat」**。它會依序：
   - **[1/2]** 把 `config/Excel/game_parameters.xlsx` 轉成 `config/CSV/game_parameters.csv`（直接讀 xlsx 位元組，Excel 開著也能轉）。
   - **[2/2]** 把 CSV 數值寫進 `js/data.js` / `js/formula.js`（含安全檢查：備份、語法檢查、失敗自動還原）。
   - 成功後視窗自動關閉；失敗會停下來顯示原因。
4. **開著的遊戲（本機）約 2 秒內會自動重新整理**，新數值立即生效——不用按任何按鈕、不用手動 F5。

> 原理：套用成功時會更新根目錄 `params_version.txt`；遊戲內的 `js/param_autoreload.js`（只在本機運作）
> 每 2 秒讀一次該檔，內容一變就自動重載。外部玩家沒有這支輪詢，也不受影響。

### 為什麼存檔不會刷新、套用才刷新？

`.vscode/settings.json` 已把整個 `config/` 資料夾（xlsx、csv）排除在 Live Server 監看之外，所以存檔不觸發重整；
只有雙擊 `.bat` 更新了 `params_version.txt`，才由遊戲內的輪詢器重整「一次」。
（改過 `.vscode/settings.json` 後，Live Server 需**停止再重新啟動**才會生效。）

### 熟悉終端機者也可打指令

```
node tools/xlsx_to_csv.cjs             # 只做 xlsx -> csv（不套用）
node tools/apply_params.cjs            # 試跑，只列出將變更，不改程式
node tools/apply_params.cjs --write    # 實際寫回（一樣會觸發自動重載）
```

## 欄位說明（哪些欄有作用）

工具**以表頭名稱定位欄位**（找 `系統分類`／`名稱`／`參數a`），不是靠欄位位置。所以：

- **`編號`、`變動` 欄只是給你看的註記,工具完全忽略**。`變動` 欄用來標記每列狀態,供 AI 判斷是否改遊戲程式（不影響 `apply_params` 套用數值）:
  - **空白 / `0`**（Excel 常把空白填成 `0`）= 原有公式,維持不動。
  - **`新增`** = 本次新增的公式;當作個人測試,AI 不會主動接進系統,等你明確說要實作。
  - **`調整`** = 你要求修改「現有公式」（擴增參數或改結構）;把具體改法寫在該列「參數化公式」與「中文說明」,AI 會據此改遊戲程式。
- 你可以**在中間插入新欄、重排欄位**,只要表頭仍有 `系統分類`／`名稱`／`參數a` 這三個名稱,就不會錯位。
- 真正對應到程式的「鑰匙」是 **`系統分類` + `名稱`** 這個組合——這兩欄的文字**不要改**,且同分類下名稱不要重複。

## ⚠️ 在 Excel 裡調數值的注意事項

改用 xlsx 當來源後，過去「Excel 存 CSV 竄改冒號 / 科學記號」的問題**已不存在**（值直接存在 xlsx 裡）。
但仍請注意：

1. **不要把格子設成「百分比」格式。** 本表的百分比是用**整數**表示（例如 25 代表 25%）。
   若把格子設成百分比再輸入 `25%`，Excel 會存成 `0.25`，套用後數值就錯了。請保持「通用/文字」格式，直接填數字。
2. **元組 / 配對格**（例如 `{1~49=25}`、`5=50`）是**文字**，照原樣打即可，別讓 Excel 自動把它當公式或日期。
   若某格開頭是 `=`、`+`、`-`、`@`，Excel 會當公式；此類格請設成「文字」格式再輸入。
3. **套用前記得先在 Excel 按存檔**：轉換讀的是「已存到硬碟」的內容，未存的編輯不算數。
4. **尾端多出來的 `0` 不用管**：Excel 常把留空的參數格自動填成 `0`。單值列本來就只讀前面幾格、不看尾端;成對/元組表（寶石商店、掉落表）工具會自動略過這些補位 `0`,不會出錯。

## 安全機制

- **試跑優先**：`apply_params.cjs` 不加 `--write` 只會列印將變更清單，不動任何檔案。
- **寫入前備份、寫入後語法檢查**：`--write` 先在記憶體備份，寫入後用 `node --check` 驗證；寫壞會**自動還原**。
- **不猜**：某參數若在程式中找不到唯一位置，工具會標為「錨點問題」並**中止寫入**。
- **恆等保證**：沒改任何值時執行，一定回報「將變更 0」。
- **轉換忠實**：`xlsx_to_csv.cjs` 產出的 CSV 與原本手工維護的格式完全一致（UTF-8 BOM、CRLF、RFC-4180 引號）。

## 進階

- 想拿別的 CSV 測試而不動正式檔：`PARAMS_CSV=/path/to/other.csv node tools/apply_params.cjs`
- 想指定別的 xlsx / 輸出位置：`PARAMS_XLSX=... PARAMS_CSV_OUT=... node tools/xlsx_to_csv.cjs`

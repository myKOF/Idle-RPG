## 1. 成長與經驗

| 項目 | 公式 | 說明 |
|---|---|---|
| 升級所需經驗 | `⌊(30 × 等級^2.2 + 40) × 轉生經驗倍率 + 升級經驗基礎增加值⌋` | 升到「下一級」所需的經驗值；等級上限 9,999。次方為 2.2。轉生經驗倍率（見轉生對照表）：0轉 1 倍，1~9 轉為 `10^轉生次數`，10轉 `10^12`。升級經驗基礎增加值於括號外相加，依轉生次數：0轉為 0，1~10轉為 `100000 × 3^(轉生次數-1)`（100000、300000、900000…1,968,300,000）。 |
| 等級基礎四維 | `5 + (等級-1) × 2` | 力量 / 敏捷 / 智力 / 耐力 四項相同，每升 1 級各 +2 |
| 技能點總預算 | 初始 `2`；未轉生時每升 1 級 `+1`，最高 `10,000` | 初始兩個 1 級技能也計入預算；已使用 = 所有技能等級總和；可用技能點 = 總預算 - 已使用。 |
| 技能裝載欄 | `clamp(⌊等級 / 20⌋, 2, 20)` | 每 20 級 +1 格，最少 2 格、最多 20 格 |

### 1.1 轉生

- 等級上限為 9,999 級（`MAX_LEVEL`）；角色達到可轉生等級 9,999 級（`REINCARNATION_LEVEL`）後可轉生，最高 10 轉；轉生後等級與當前經驗重置為 1 級／0 經驗。（等級上限與可轉生等級為兩個獨立參數，目前皆為 9,999。）
- 轉生保留裝備、技能、資源與關卡進度。
- 階級依序為冒險者、勇者、大劍師、破世者、不朽者、王者、大主宰、神聖尊者、大聖王、至高主宰、位面創世神。
- 生命、法力與四維在原始總值完成後套用最終倍率：1～10 轉為 ×10、×20、×40、×80、×160、×320、×640、×1280、×2560、×5120。
- 攻防「定值」的轉生強化：物攻/魔攻/物防/魔防的定值加成，於基底括號內另加 `係數×定值×指數底^轉生次數`（物攻/魔攻：1.2×定值×2.8^轉生；物防/魔防：0.75×定值×2.7^轉生），詳見 §2.2。
- 轉生後升級經驗需求倍率：1~9 轉為 `10^轉生次數`（每轉 ×10），10 轉為 `10^12`（依轉生對照表）。
- 玩家升級經驗需求：`xpForLevel(l) = floor((30 × l^2.2 + 40) × 轉生經驗倍率 + 升級經驗基礎增加值)`，其中 l 為目前等級；升級經驗基礎增加值依轉生次數（0轉為 0，1~10轉為 `100000 × 3^(轉生次數-1)`）於括號外相加。
- 轉生後每升一級獲得 1 點轉生天賦點；技能點不再增加。轉生天賦點用於 1～5 轉天賦，潛力則是新的技能分類，直接消耗既有技能點，不另設潛力點。
- 一般技能上限為 `20 + 轉生次數 × 10`（被動技能基礎 30）；融合技能在其原始上限上再加 `轉生次數 × 20`。
- 神鑄系統需**同時滿足兩個條件**才開放：角色等級 ≥ `FORGE_UNLOCK_LEVEL`（解鎖等級）**且**轉生次數 ≥ `FORGE_UNLOCK_REINCARNATION`（解鎖轉生次數）；開放後永久保留，轉生不會關閉。

## 2. 玩家屬性（50+ 屬性派生）

### 2.1 彙總流程

```
等級基礎四維
  → 累加：裝備詞條（× 強化倍率）＋ 被動技能（每級效果 × 技能等級）
        ＋ 鑲嵌寶石（× (1 + 寶石鑲嵌效率%)）＋ 附魔（元素附傷 / 抗性 / 功能）
  → 派生出面板上所有數值（多數有上限）
```

- 詞條「攻擊速度%」計入攻速加成桶；「火焰抗性%」等六系計入元素抗性桶。
- **強化倍率**：裝備每 +1，其所有詞條數值 ×(1 + 0.05×強化等級)。
- 融合寶石（雙屬性）逐屬性計入，同樣吃寶石鑲嵌效率放大。
- **屬性面板預覽（三套裝備）**：側欄屬性面板顯示「檢視中」裝備套的屬性——切到別套檢視（尚未按確定切換）時，面板即以該套計算 would-be 屬性並顯示「👁 屬性預覽」提示；戰鬥、回復、掉落等一切遊戲邏輯仍以「穿著中」那套（`getStats()`）為準，切頁不影響實際數值。

### 2.2 基礎屬性派生公式

| 屬性 | 公式 |
|---|---|
| 生命上限 | `(120 + (等級-1)×22 + 耐力×10 + 生命定值) × (1 + 生命%)` |
| 生命恢復/秒 | 詞條/寶石加成值（另有野外基礎回復，見 §3.5） |
| 法力上限 | `40 + 智力×2 + 法力定值` |
| 法力恢復/秒 | `2 + 智力×0.002 + 加成` |
| 物理攻擊 | `(8 + 物攻定值 + 1.2×物攻定值×2.8^轉生次數 + 力量×1) × (1 + 物攻%)` |
| 魔法攻擊 | `(6 + 魔攻定值 + 1.2×魔攻定值×2.8^轉生次數 + 智力×1) × (1 + 魔攻%)` |
| 物理防禦 | `(3 + 物防定值 + 0.75×物防定值×2.7^轉生次數 + 力量×0.35 + 耐力×0.65) × (1 + 物防%) × (1 + 天賦物防%［物防鍛體/重甲共鳴，獨立乘區］)` |
| 魔法防禦 | `(2 + 魔防定值 + 0.75×魔防定值×2.7^轉生次數 + 智力×0.35 + 耐力×0.65) × (1 + 魔防%) × (1 + 天賦魔防%［魔防鍛體/魔鎧共鳴，獨立乘區］)` |

> 攻防四項皆已**不再隨等級直接成長**（僅透過四維主屬性間接成長）；「定值」除了直接相加外，另有 `係數×定值×指數底^轉生次數` 的轉生強化項（0 轉時指數為 1，即定值×係數的固定加成）。物防%與魔防%為**各自獨立**的百分比桶：物防%來源＝裝備「物理防禦%」詞條、神壁、鋼鐵之膚；魔防%來源＝神壁、鋼鐵之膚（裝備詞條池目前沒有「魔法防禦%」詞條）；天賦物防%/魔防%另為獨立乘區。

## 3. 戰鬥核心

### 3.1 防禦減傷（核心公式）

```
減傷率 = 防禦 / (防禦 + 60 + 8 × 攻擊者等級)
```

防禦收益遞減；攻擊者等級越高，同樣防禦的減傷率越低。當有效防禦 `≤ 0` 時，減傷率直接視為 `0%`，等同於不再享有防禦減傷。

### 3.1.1 抗性減傷

物理抗性、魔法抗性各自使用獨立的抗性減傷參數；六系元素抗性共用元素抗性減傷參數。三組參數目前皆為 `次方=1.8、常數=10、敵人等級係數=0.1`，可在參數表「3-戰鬥核心」分別調整。抗性不設上限，抗性小於等於 0 時減傷率為 0%。

`減傷率 = (抗性總合^次方) / (抗性總合^次方 + 常數 + 敵人等級×敵人等級係數)`

技能或減益造成的破防先計算有效防禦：

`有效防禦 = 防禦 × (1 - 技能防禦降低% / 100) × (1 - 破甲%) × (1 - 穿透%)`

實際程式中的技能防禦降低會以 `防禦 × (1 - 防禦降低% / 100)` 計算；當降低超過 100% 時會得到負防禦，但後續仍依 `有效防禦 ≤ 0` 處理為 0% 減傷，不會產生額外負防禦傷害。

### 3.2 單次攻擊結算順序（resolveHit）

```
1. 命中判定：命中機率 = clamp(攻擊者命中% - 防守者閃避%, 5%, 100%)，未命中 → MISS
2. 基礎傷害：
   物理部分 = 物攻 × (1 - 防禦減傷(有效物防)) × (1 - 物理抗性減傷率)
   魔法部分 = 魔攻 × (1 - 防禦減傷(有效魔防)) × (1 - 魔法抗性減傷率)
   其中 有效防禦 = 防禦 × (1 - 破甲%) × (1 - 穿透%)
   ※ 玩家普攻為「雙修型」：物理+魔法兩部分同時結算相加
3. 傷害浮動：× rnd(0.9, 1.1)
4. 暴擊判定：機率 = 暴擊率，成功 → × 暴擊傷害%（基礎 150%）
5. 元素附加：每系元素值 = 固定值元素攻擊 + 附傷基底 × 元素附傷%（附傷基底 = 步驟 2~4 完成後的當次傷害，故附傷跟隨防禦減傷、浮動與暴擊縮放）
   每系傷害 = 每系元素值 × (1 - 對應元素抗性減傷率)，不重複套用魔法抗性，並觸發元素特效（§3.3）
6. 真實傷害：+ 攻擊力 × 真傷%（無視防禦、抗性與格擋）
7. 敵種傷害加成分為兩個獨立乘區：對菁英先乘既有裝備／詞條的 `(1+對菁英傷害%/100)`，再獨立乘 4 轉「破菁法則」天賦的 `(1+天賦對菁英傷害%/100)`；對 BOSS 使用「弒王法則」、對普通敵人（非菁英且非 BOSS）使用「清場法則」，規則相同。例如既有 +10% 且天賦 +1%，結果為原始傷害 ×1.10×1.01，而不是 ×1.11。
8. 格擋判定：機率 = 格擋率（上限由 `STAT_CAPS.blockRate` 控制，預設 50%），成功 → 傷害 × (1 - (30% + 格擋減傷%))，總減傷上限由 `30 + STAT_CAPS.blockDmgRed` 控制（預設 80%）
9. 神鑄特效「聖佑」：受到的所有傷害再按比例減免，上限 50%
10. 全局減傷：減傷率 = `min(GLOBAL_DMG_RED_CAP%, 全局減傷總合 / (全局減傷總合 + GLOBAL_DMG_RED_DENOMINATOR))`，套用 `傷害 × (1 − 減傷率)`，在最低傷害與護盾結算前套用
11. 敵種傷害抗性：依攻擊者敵種（普通/菁英/BOSS）選用防守方對應的「普通敵人/菁英/BOSS傷害抗性」總合，減傷率 = `抗性值總合 / (抗性值總合 + 100 + 0.25 × 攻擊者等級)`（`ENEMY_TYPE_DMG_RED_A/B`），套用 `傷害 × (1 − 減傷率)`；為傷害公式最末端減傷（全局減傷之後、最低傷害之前）
12. 最低傷害 1 → 護盾優先吸收 → 扣血
13. 反震：攻擊方受到 防守方最大生命 × 反震% 的傷害（最低 1）
```

補充：一般物理/魔法傷害、元素附加傷害與持續傷害會依各自流程套用全局減傷；技能類型為「真實」的技能目前走直接扣血路徑，傷害為 `技能原始傷害 × rnd(0.95, 1.05)`，不經防禦、抗性、格擋與 `resolveHit` 的全局減傷流程。被動「真傷」則是在 `resolveHit` 中附加，會跟隨該流程。

**連擊數（暴擊率破 100% 衍生的多段攻擊）**：見 §2 衍生表「連擊數」＝ `comboHitsFor(暴擊率)`。每次出手先以 `rollComboHits` 擲出本次追加次數（整數固定追加、小數部分依機率再追加一次，例：2.57 → 額外 2 次＋57% 機率第 3 次）：
- **普攻**（`doPlayerAttack`）：主攻擊命中且未一擊致死後，對同一目標追加該次數的完整普攻（每段各自結算命中/暴擊/特效；僅主攻擊觸發，遞迴段不再疊連擊，與「連擊被動」互不遞迴）。
- **技能**（`castSkill`）：僅將技能的「直接傷害段」整段重複追加該次數；DoT／減益／吸血／治療等在傷害迴圈外一次結算，故**持續傷害不受連擊數影響**。
- 連擊數為玩家專屬衍生屬性，怪物/BOSS 不套用。

**技能命中率**：技能傷害段的命中率＝`max(100%, 玩家命中率)`（`neverMiss` 技能則固定 999% 視同必中）。即技能至少享有 100% 基礎命中（低命中期不受影響），命中率高於 100% 時亦能壓過高閃避敵人（沿用 §3 步驟 1 的 `clamp(命中 − 閃避, 5%, 100%)`）。※ 野外怪物閃避隨階段成長（§4.1 `10% + 階段×4`），高階敵人閃避可達數百％，故技能需要命中率加持才不會被夾到 5% 命中下限而大量 MISS。

### 3.4 狀態效果

- **減速**：攻速 ×0.7（-30%），即攻擊冷卻累積速度打七折。
- **暈眩**：期間完全無法行動（普攻與技能皆停）。
- **控制抵抗**：`ctrlRes%` 機率完全免疫本次暈眩/減速。
- **受控時間** = 原始時間 × (1 - 韌性%) × (1 - 控制時間縮減%)。
- **控場遞減（對敵方，2026-07-17 新增）**：對敵方施加會改變攻擊頻率的控制（暈眩/減速/攻速類減益）時，`實際持續時間 = 原持續 × max(0, 1 − 敵人存活秒數 × 每秒遞減%)`。普通敵人每秒 −1%（例：8 秒暈眩在戰鬥 50 秒時施放剩 4 秒，100 秒後完全免疫）、菁英每秒 −3%（約 33 秒後免疫）、BOSS 維持完全免疫（既有規則）。玩家受控不受此遞減影響。常數 `CONTROL_DECAY_PER_SEC_NORMAL/ELITE`（formula.js §3，`controlDurationFactor`；套用點＝`applyEffect`/`applyBuff`，所有控制來源——技能、冰元素特效、暈眩被動——皆生效）；戰鬥日誌與冰緩觸發顯示實際套用結果（歸零時不再誤報）。
- **持續傷害（DoT）**：同名效果疊加取高（傷害與時間各取較大者）；中毒為獨立通道，其餘（流血/燃燒/詛咒/侵蝕）共用列表，全部無視防禦。

### 3.5 回復與護盾

| 項目 | 公式 |
|---|---|
| 野外基礎回復 | 每秒回復 `最大生命 × 1.5% + 生命恢復屬性`（高塔內只有生命恢復屬性與再生增益，無 1.5% 基礎） |
| 擊殺回復 | 野外每擊殺一隻回復 `最大生命 × 12%`（吸魂被動再另加 `最大生命 × 吸魂%`） |
| 溢出轉護盾 | 治療溢出部分的 **1%** 轉為護盾（`SHIELD_OVERFLOW_PCT`） |
| 護盾上限（治療轉化） | `最大生命 × 10% × (1 + 護盾效率%)`（`SHIELD_HEAL_CAP_PCT`）；只限制溢出治療新增的護盾，不會降低既有更高護盾。 |
| 護盾技能加成 | 目前護盾基準 × `(1 + 技能護盾% × (1 + 護盾效率%) / 100)`；若目前無護盾，改以最大生命作為基準。同一護盾技能狀態不可疊加，重放只刷新到該基準換算後的護盾值。技能給予的護盾另有上限 `最大生命 × SHIELD_SKILL_CAP_PCT%`（目前 10000%＝100 倍生命；參數表「3-戰鬥核心／護盾上限(技能給予)」）。 |
| 護盾條滿值 | 每次護盾增加後，以當前護盾值作為新的滿條基準；受傷只扣目前護盾，護盾歸零時清除基準。 |
| 吸血 / 吸魔 | 每次造成傷害回復 `傷害 × 吸血%` 生命 / `傷害 × 吸魔%` 法力 |

### 3.6 怪物攻擊玩家

- 怪物固定：暴擊率 5%、暴擊傷害 150%。命中率不再固定 100%，改隨敵人等級（野外）或樓層（BOSS）成長，詳見 §4.1、§4.3；命中判定沿用 §3「clamp(攻擊者命中 − 防守者閃避, 5%, 100%)」。
- `magic: true` 的怪物走**魔法傷害**（對玩家魔防），其餘走物理。
- 野外怪物初次出手延遲：`1/攻速` 秒（與玩家相同，無額外延遲）。
- 普通關卡若同時出現多隻敵人，敵人各自獨立攻擊；玩家普攻固定鎖定第一隻存活敵人，該敵人死亡後才切換下一隻。
- 玩家死亡：目前階段 -10，最低退到第 1 階段，3 秒後復活（滿血滿藍、狀態清空）。

## 6. 裝備數值

### 6.1 生成規則

- 部位：隨機 10 種之一；等級 = `階段 ± 1`（最低 1）。
- 詞條數：依稀有度表（普通 1~2 → 創世 7）；玩家有「詞條上限率」時，`詞條上限率%` 機率 +1 條（總上限 8 條）。
- 詞條種類：依詞條池權重抽取（高階詞條有最低稀有度 `minR`、部位限定 `slots` 門檻），不重複。
- 幸運值：每條詞條 `幸運/2%` 機率重骰一次數值取較佳值。
- 稀有級（含）以上附帶 1 個特殊被動。

### 6.3 太古詞條

- 只有等級 `>=200`、史詩級以上裝備會在生成時判定太古詞條；普通、精良、稀有裝備不會出現太古詞條。自然掉落使用來源敵人的規則；神鑄成功產物則使用一般敵人的規則，並以素材最高等級作為敵人等級。合成與 GM 直接生成仍不套用自然掉落判定。
- 野外每個詞條獨立判定：`min(3%, 1% + (敵人等級-250)×0.1%)`；敵人等級 `<250` 時維持基礎 1%。
- 高塔 BOSS 每個詞條獨立判定：樓層 `>=40` 時為 `min(100%, 5% + (樓層-40)×0.5%)`（同 §4.3，以樓層計算）。
- 洗煉時也只有史詩級以上裝備可產生太古詞條；太古裝備再次洗煉仍可能回到普通詞條。
- 太古詞條數值固定為原洗煉上限（基準值 × 1.2）再乘 `1.35`；也就是原詞條上限的 135% 強度。
- 顯示時詞條名稱、數值與前方標記皆使用橘色，標記為較大的六角星 `✡`。

### 6.4 詞條數值

```
基準值 = (base + base × lv係數 × (裝備等級 - 1)) × 稀有度倍率
實際值 = 基準值 × rnd(0.8, 1.2)      ← 即洗煉區間為基準值的 80% ~ 120%
```

- `base`、`lv係數` 見 `data.js` 的 `AFFIX_POOL`；稀有度倍率：普通 1.0 / 精良 1.35 / 稀有 1.75 / 獨特 2.3 / 史詩 3.0 / 傳說 4.0 / 神話 5.2 / 創世 6.8 / 神鑄創世 10.2。
- **全局減傷詞綴**：定值型 `base=3、lv係數=0.35`，最低史詩稀有度，無部位限制，因此所有裝備部位均可出現；多件裝備直接加總。
- **對普通敵人傷害%詞綴**：百分比型 `base=3、lv係數=0.035`，最低獨特稀有度、無部位限制；僅對非菁英且非 BOSS 的普通敵人生效（§3.2 步驟 7）。基礎值恢復為原先的 10 倍，成長係數不放大。
- **既有存檔特例**：`normalDmgAffixScaleV2` 回復曾套用 V1 的低值；V3 修復仍低於新基準的值；V4 將先前把成長係數誤放大 10 倍的值除回 10，三者均為一次性且不重複執行。
- **敵種傷害抗性詞綴（普通敵人傷害抗性/菁英傷害抗性/BOSS傷害抗性）**：定值型 `base=6、lv係數=0.35`（放出量約為物理防禦詞條的 2 倍），最低獨特稀有度、無部位限制；多件裝備直接加總，減傷公式與套用時機見 §2.5 與 §3.2 步驟 11。
- **獨特級詞綴門檻（2026-07-16 調整）**：對普通敵人傷害%、對菁英傷害%、對BOSS傷害%、三種敵種傷害抗性、掉寶率%、經驗加成%、金幣加成% 共 9 種詞條的最低稀有度一律為**獨特**（`minR=3`），僅獨特（含）以上裝備會出現。

### 6.5 特殊被動數值

```
被動數值 = base + perR × (稀有度 - 2)      ← 稀有=基準，每高一階 +perR
```

| 被動 | base / perR | 上限（多件疊加後） |
|---|---|---|
| 破甲 | 10 / 2 | — |
| 反震 | 5 / 1 | — |
| 連擊 | 10 / 2 | 45% |
| 暈眩 | 6 / 1.5 | 30% |
| 減速 | 12 / 3 | — |
| 真傷 | 6 / 1.5 | — |
| 吸魂 | 5 / 1.5 | — |

神鑄創世特效「神力」不直接加總到物理／魔法攻擊百分比，而是作為額外乘區：
`攻擊基礎 × (1 + 原有攻擊% / 100) × (1 + 神力% / 100)`；多件神力效果彼此相乘。

### 6.6 附魔數值

```
攻擊類（元素附傷）= (5 + 裝備等級×1.2) × 稀有度倍率 × (1 + 0.15×寶石等級)
                     火焰附魔再 ×1.25（定位：純高額傷害）
防禦/功能類（%）  = min(8 + 稀有度×4 + 寶石等級×3, 60)
```

- 附魔欄位數依稀有度：普~稀有 1、獨特~傳說 2、神話/創世 3。
- 手動附魔費用：1 本書 + 5 附魔精華；取下返還書、不退精華。
- 「寶石等級」參數只在生產線混合合成時帶入（= 消耗寶石等級 × (1+寶石鑲嵌效率%)），手動附魔為 0。

### 6.7 裝備戰力評分（自動換裝依據）

```
評分 = Σ(詞條值 × 權重 × 強化倍率)
     + Σ(鑲嵌寶石值 × 對應詞條權重)
     → 有特殊被動 ×1.15
     + Σ(附魔值 × 1.2［攻擊類］ or × 2［防/功能類］)
     → × (1 + 稀有度 × 0.06)
```

權重表 `SCORE_WEIGHTS` 見 `formula.js` §6（例：攻速 2.8、物攻% 2.6、暴擊率 2.4、生命定值僅 0.16）。
自動換裝條件：新裝評分 > 舊裝評分 × 1.02（避免頻繁換裝）。

### 6.8 分解產出

```
碎片 = max(1, (2 + 裝備等級×0.6) × 分解係數 × rnd(0.85, 1.15))
金幣 = (3 + 裝備等級) × 分解係數 × 0.5
附魔精華 = 依裝備稀有度基礎機率 × (1 + 精粹透鏡加成總合/100) 結算
鑲嵌的寶石（含融合寶石）分解前自動取回；分解本身不會產出寶石
```

分解係數（`RARITIES.salv`）：普通 1.0 → 精良 1.7 → 稀有 2.8 → 獨特 4.5 → 史詩 7.5 → 傳說 12 → 神話 19 → 創世 30 → 神鑄創世 45。

拆解取得太古精華：史詩 0.5%、傳說 1%、神話 10%、創世 100%、神鑄創世 100%，每件裝備獨立判定 1 顆。

## 8. 寶石

### 8.1 能力數值（超線性成長）

```
數值 = base × 等級 × (1 + 0.2 × (等級-1))
```

即等級倍率為：Lv1 ×1、Lv2 ×2.4、Lv3 ×4.2、Lv4 ×6.4、Lv5 ×9。`base` 見 `data.js` 的 `GEM_TYPES`（**18 種**：12 攻擊/功能 + 6 防禦〔翡翠=韌性、綠松石=格擋率、瑪瑙=格擋減傷、珍珠=護盾效率、孔雀石=物理抗性、螢石=魔法抗性〕，2026-07-09 新增）。

## 9. 技能

### 9.1 基本數值

> 📈 2026-07-09 傷害重調：全部傷害技能的 base/per **×2.0 ~ ×3.0**（越強的技能加越多：
> 低耗速射 +100%、中型爆發 +120~150%、大招如隕石術/狂暴打擊 +180~200%）；
> 各技能實際數值見 `skills.js` 的 `SKILLS` 表。

| 項目 | 公式 |
|---|---|
| 技能原始傷害 | `(base + per × (等級-1))% × 對應攻擊力（物攻或魔攻）` |
| 一般技能法力消耗 | `round(原始消耗 × (1 + 0.1 × (技能等級-1)))`；非複利，例：50 → 55 → 60 |
| 多敵人技能傷害 | 當目標數 `N>1`：`原始傷害 × (1 + 範圍傷害%) ÷ N`；單一目標不套用範圍傷害加成，也不除以目標數 |
| 多段技能 | 上式為**每段**傷害，段數 `hits` 段各自結算（暴擊、閃避獨立判定） |
| 普攻目標 | 一次只攻擊一名敵人，不套用範圍傷害；目前鎖定第一名存活敵人，死亡後才切換 |
| 實際冷卻 | `技能冷卻 × (1 - 冷卻縮減%)` |
| 施放硬直 | `0.5 秒 × (1 - 施法速度%)`（延後下一次普攻） |
| 技能共用冷卻(GCD) | `0.4 秒`（固定值，不受冷卻縮減影響） |
| 升級費用 | `⌊20000 × 當前等級 + 20^(1 + 當前等級/10)⌋` 金幣（上限 5,000,000；降級全額退還該級費用） |
| 等級上限 | 一般主動 `20 + 轉生次數×10`；被動 `30 + 轉生次數×10`；融合技 `素材等級加總 + 20 + 轉生次數×20` |
| 里程碑 | Lv.4 解鎖附加效果、Lv.8 強化該效果（覆蓋式，詳見 `skills.js` 的 `UNLOCKS` 表） |
| 技能樹門檻 | 第 N 階技能需先在該系投入 `N × 3` 點 |
| buff/治療縮放 | 皆為 `base + per × (等級-1)` 形式 |

| 舊存檔融合技 | 讀檔時移除 fx 快照改為動態重算（`fusionFxDynamicV1`，見 `ONE_TIME_MIGRATIONS.md`）；素材技能已不存在者保留快照作後備。＊註：舊 `skillDmgV2` 等比加成旗標已被動態重算取代——自現行（已調升）SKILLS 定義重算即直接得到調整後數值，無需再對快照加成 |

| 初始等級 | 素材等級加總（素材歸零、點數轉移；刪除融合技全額歸還） |
| 變異觸發率 | `45% + 幸運值/3`（8 種變異，需與融合結果相關，至多 1 種） |

## 10. 天賦與潛力

- 天賦於完成 1 轉後開放；目前實作 1～5 轉，每轉 8 個節點，每個節點最高 100 級。
- 天賦升級成本為目標等級：`Lv.1 = 1`、`Lv.2 = 2`，因此升到 Lv.`L` 的總成本為 `L × (L + 1) / 2`。降級退還目前等級成本，歸零退還累計成本。
- 節點效果：Lv.1～50 每級使用 `low`；Lv.51～100 每級使用 `high`。該轉 8 個節點全部 Lv.100 時，該轉天賦效果乘 2。
- 2 轉前六個節點（烈焰/寒霜/雷霆/毒脈/聖輝/暗影共鳴）＝元素附傷%：攻擊時附加「當次傷害 × 天賦%」的對應元素傷害（例：1% 火附傷、當次傷害 10000 → 額外 100 火傷）；六系並存時各自結算、各吃對應元素抗性並獨立觸發 §3.3 元素特效。普攻與技能直接傷害段皆適用；真實傷害技能走直接扣血路徑，不附加。每級數值：Lv.1～50 每級 0.25%、Lv.51～100 每級 0.5%（滿級 37.5%；該轉全滿 ×2 → 75%）。
- 「傷害偏折」（2 轉）與「絕對偏折」（4 轉）＝全局減傷額外提高%（**乘算**）：全局減傷總值 = 其他來源加總 × (1 + 兩天賦%合計/100)；沒有其他全局減傷來源時天賦不憑空提供定值。每級數值：Lv.1～50 每級 0.5%、Lv.51～100 每級 1%（滿級 75%；該轉全滿 ×2 → 150%。例：裝備 214k、傷害偏折全滿 ×2 = +150% → 535k）。
- 「物防鍛體/魔防鍛體」（1 轉）與「重甲共鳴/魔鎧共鳴」（4 轉）＝物理/魔法防禦額外提高%（**獨立乘區**）：物防 × (1 + 物防天賦%合計/100)、魔防 × (1 + 魔防天賦%合計/100)，與防禦公式共用的物攻%/魔攻%桶連乘、不相加；物/魔各自獨立。（裝備「物理防禦%」詞條已不參與防禦公式，見 §2.2。）
- 5 轉「武技/法術/守護/奇策/被動昇華」＝對應類別技能效果 ×(1 + 天賦%/100)（`talentSkillEffectMultiplier`）。作用範圍＝該技能的**全部效果**：直接傷害（含衍生 DoT 與元素占比）、治療、護盾、持續再生、增益/減益數值、死亡詛咒 DoT、金幣（點金手）、法力回復；被動昇華作用於被動技能的屬性數值（`被動數值 = 每級效果 × 技能等級 × (1 + 被動昇華%)`，computeStats）。暈眩/減速等**時間類**與淨化不受倍率影響。**融合技**倍率 = 素材技能類別倍率的**平均**（`skillEffectTalentMultiplier`；舊存檔無素材記錄的融合快照 = 1）。技能面板說明顯示基礎值，戰鬥浮字與日誌顯示套用倍率後的實際值。
- 4 轉「清場法則／破菁法則／弒王法則」分別只對普通敵人／菁英／BOSS 生效；各自的天賦百分比與對應裝備／詞條百分比為**獨立乘區**，不會先相加成單一敵種傷害百分比。
- 「潛力啟示」（4 轉）與「潛力覺醒」（5 轉）目前暫停開放升級，天賦節點本身置灰且不可升級；原定投入 1 級固定解鎖潛力技能的規則保留，重新開放時沿用。兩個節點各依 Lv.1～50 每級給 2 點技能點、Lv.51～100 每級給 4 點，單一節點滿級共 300 點，合計併入一般技能點預算。
- 其他天賦依轉數鎖定；4／5 轉潛力節點解鎖後，依序開放前 5 個潛力技能。
- 潛力以技能分類管理，最高等級沿用一般技能上限：`20 + 轉生次數 × 10`；升級消耗既有技能點，並沿用技能升級金幣公式 `min(5,000,000, floor(20,000 × Lv + 20^(1 + Lv/10)))`。
- 已實作的潛力效果包含冷卻縮減、一次性致命復生、掉落收益、背包容量、元素附加傷害乘算提高（元素核心：所有元素附傷%與固定值元素攻擊 × (1 + 2% × 等級)，未點的元素不會憑空附傷）、低血量處決增傷、護盾與法力循環、高塔限時與離線收益加成；節點資料集中於 `js/data.js` 的 `POTENTIAL_TALENTS`。目前 5 轉新增的「虛空背包」與「元素核心」暫時置灰並禁止升級，待潛力系統問題修正後再開放；天賦本身為正式公開功能，不受本機 hostname 限制。

## 11. 離線收益（2026-07-17 改版：固定速率獵殺菁英怪＋逐殺掉落）

```
有效離線時間 = min(實際離線秒數, OFFLINE_MAX_HOURS=24 小時)（1 分鐘內不計）
計算等級    = max(1, ⌊(目前地圖最高階段 − OFFLINE_LEVEL_REDUCE=10) / 10⌋ × 10)
              例：沼澤最高 256 → 256−10=246 → 捨去個位數 → 240 級「沼澤菁英怪」
擊殺數      = ⌊有效離線秒數 / OFFLINE_KILL_INTERVAL=20 × (1 + 潛力「離線預言」%)⌋
              （固定每 20 秒擊殺 1 隻計算等級的當前場景菁英怪）

金幣 = round(菁英怪金幣 × 場景倍率 × (1 + 金幣加成%)) × 擊殺數
經驗 = round(菁英怪經驗 × 場景倍率 × (1 + 經驗加成%)) × 擊殺數
掉落 = 每一隻菁英怪單獨擲骰，與野外擊殺共用掉落表與倍率（菁英掉落倍率 ELITE_DROP_MULT、
       掉寶率、場景倍率）：裝備（各品質獨立擲骰 → 送入熔爐佇列；滿載折算碎片
       每件 3 × 分解係數）、寶石、附魔書（階段 8+）、太古精華（250 級+）、魔塵（150 級+）、
       自動機組零件（階段 5+）
```

上線時彈出「離線收益」確認界面（`#offline-modal`）：離線時長、擊殺數（等級＋場景＋菁英）、
經驗、金幣、掉落明細列表（裝備以品質×數量彙總，例：傳說裝備×100）；收益於計算時即入帳，
彈窗為確認展示。相關參數集中於參數表「10-離線」段（有效離線時間／計算等級／擊殺速率）。

## 12. 熔爐（正式版：品質勾選路由）

原「新熔爐（測試版）」已**合併取代**舊生產線的輸送帶/篩選節點/分解槽，**全服開放**（本地服限定已解除）。掉落裝備一律經 `pushConveyor` 進入**總佇列**；佇列「總量」（總佇列＋各爐專屬佇列合計）達 `NEW_FORGE_QUEUE_CAP = 20,000` 時新裝備丟棄（同舊輸送帶滿載規則，並防存檔膨脹）。合成節點維持關閉。

**改版遷移**（`migrateSave` / `sanitizeNewForge`）：舊存檔載入時——舊輸送帶滯留裝備自動併入熔爐佇列、舊分解槽零件安裝解除（零件釋回零件庫）、專屬材料計數（`forgeMats`）與導入開關欄位移除；並顯示一次性改版公告彈窗（`#forge-rebuild-modal`），熔爐頁籤高亮閃爍（`.nf-glow`）至玩家切到該分頁為止。

### 12.1 熔爐與品質路由

- **可設熔爐數與轉生連動**：`newForgeMaxFurnaces = clamp(2 + 轉生次數, 2, 12)`（0 轉 2 座、每 1 轉 +1 座、硬上限 `NEW_FORGE_MAX = 12`；`formula.js`）。熔爐大圖統一 `images/furnace_LV1.png`。
- **每座熔爐＝單一傳送帶＋品質勾選（圖2）**：勾選品質的裝備自動進入該熔爐傳送帶拆解；未勾選＝保留入包。上鎖與神鑄創世永遠保留。預設勾選：普通~傳說。每爐可獨立「啟用」暫停。
- **專屬佇列**：每座熔爐擁有自己的專屬佇列（`fu.queue`，上限 `NEW_FORGE_FURNACE_QUEUE_CAP = 9,999`/爐）。頂部「佇列」＝**總佇列**（尚未派發的剩餘總量，派發後遞減）；帶尾「+N」＝該爐專屬佇列**真實件數**（各爐獨立、非共用計數——各爐加總＋總佇列＝全部待處理量）。
- **派發**：每 `NEW_FORGE_INTERVAL = 2` 秒一輪、每輪最多 `30 × 熔爐數` 件（`NEW_FORGE_ROUTE_PER_TICK`）——總佇列逐件（先進先出）分派到啟用且勾選該品質、專屬佇列未滿的熔爐中「**帶＋專屬佇列負載最少**」者（平均分流）；無符合熔爐→保留入包；符合者專屬佇列皆滿→留總佇列等待。傳送帶（30 件）自該爐專屬佇列隨時補位。更強自動換裝開關位於熔爐頁頂部（仍存於 `G.factory.autoEquip`）。
- **一致性**：品質取消勾選／停用熔爐 → 專屬佇列中不符項退回總佇列重新派發（帶上裝備視為已承諾不回退）；移除熔爐（或轉生上限裁減）→ 專屬佇列回總佇列、帶上裝備退回背包。
- **入爐**：每座熔爐每 2 秒拆解帶頭 1 件；鑲嵌寶石（含融合寶石）拆解前自動取回。
- 佇列顯示**完整數字**（`fmtFull`，不簡寫）。帶尾 +N 顯示封頂 +9999（＝專屬佇列上限）、tooltip 為精確數。

### 12.2 拆解產出（沿用舊分解槽規則）

專屬材料系統已刪除。入爐拆解走 `factory.js` `doSalvage`（公式見 §7）：**碎片、金幣、附魔精華、太古精華**與各零件事件（翻倍/大豐收/回收書/經驗/掉零件），零件加成改以「**該熔爐零件格**」的快照計算（`newForgePartBonus(fu, key)`）。手動「一鍵分解」（裝備分頁）走無零件加成的基礎產出。

### 12.3 零件置入格

- 每座熔爐初始 **3 格**（`NEW_FORGE_PART_SLOTS_INITIAL`），金幣逐格解鎖至 **8 格**（`NEW_FORGE_PART_SLOTS_MAX = 8`）。
- 解鎖金幣：`50000 × 轉生次數² + 10000 × (該熔爐已解鎖格數 − 1)^(4 + 熔爐數量)`（`formula.js` `newForgePartSlotCost`；係數 `data.js` `NEW_FORGE_SLOT_COST_*`）。例：0 轉、2 座熔爐、第 4 格 = 10000×2⁶ = 640,000。
- 零件裝配（**完全自由**）：點擊空零件格開啟零件列表（該熔爐下方、依類型分組），點擊晶片依類型裝配「持有的最高階零件」**數值快照**——不佔用、不消耗零件庫存，同類型可重複裝滿、多座熔爐可共用，僅零件格數為上限；點擊已裝格卸下（快照丟棄）。快照為裝配當下數值，撿到更高階後重新裝配即可更新。
- 零件效果：**全部 10 種分解槽零件皆對該熔爐生效**——加速齒輪＝入爐速度倍率 `1 + Σ(數值+固定加成)/100`；其餘（碎片熔煉爐/淘金濾網/精粹透鏡/太古精華萃取器/拓本回收臂/複製處理艙/知識回收器/探礦核心/幸運晶片）經 `doSalvage` 套用，效果同 §7.3。
